1. xml 선언
  --> xml 선언시  ?와 xml 키워드에 공백을 사용하면 안된다.
  --> 문서의 제일 첫번째 작성을 원칙으로 한다.

  ex) <? xml --> error

  version : 1.0의 값을 가진다. (필수)
  encoding : 한글 사용시 'euc-kr' 또는 ksc5601 등을 사용한다. (선택)
  standalone : DTD, Schema문서와 같이 실행할 것인지 여부 (선택)

2. element(태그) 작성법
  (1) 시작태그와 끝태그를 가져야 한다.
  (2) 시작태그과 끝태그의 이름은 같다.
  (3) 시작단어로 숫자나 특수문자를 사용할 수 없고
  (4) 이름에 공백을 허용하지 않는다.
  (5) 중첩해서 작성될 수 없다.
  (6) 이름은 대소문자를 구분한다.
  (7) 이름의 시작을 xml로 할 수 없다.
  (8) root element는 반드시 하나만 존재한다.

3. element 사이에 올 수 있는 형태들
  (1) 하위의 element
  (2) 일반 문자
  (3) 빌트인 Entity
     * < : &lt;
     * & : &amp;
     * > : &gt;
     * " : &quot;
     * ' : &apos;
  (4)  파싱(해석)을 하지 않는 CDATA Section
    ex) <![CDATA[ 문자열 ]]>

4. DTD
  (1) 종류
     1) 내부 DTD
        --> 문서 유형에 대한 선언이 xml문서에 존재
 --> 하나의 xml문서에서만 사용됨

     2) 외부 DTD
        --> 독자적인 파일로 존재하는 dtd문서
 --> 여러 xml파일에서 참조 가능

  (2) Element 선언 문법
      <!ELEMENT  엘리멘트이름  (컨텐츠 유형)>

      1) Element의 컨텐츠로 문자를 갖는 형태
         <!ELEMENT  exam  (#PCDATA)>

      2) Element의 컨텐츠로 하위 Element를 갖는 형태
         <!ELEMENT  exam  (자식1,  자식2,  자식3,.....)>

  * 출현 빈도수에 대한 표현
  a. , : 순차적인 출현
  b. | : A, B중 선택적인 출현
  c. ? : 0~1번 출현
  d. + : 1 ~ 무한대 출현
  e. * : 0 ~ 무한대 출현
  f. 기호없음 : 반드시 한번 출현
  
      3) Element의 컨텐츠를 허용하지 않을 경우
         <!ELEMENT exam  EMPTY>

      4) Element의 컨텐츠중 문자데이터와 자식 Element를 혼합할 경우
         <!ELEMENT  exam  (#PCDATA | 자식)*>

      5) Element의 컨텐츠로 dtd에 선언된 모든 데이터 및 엘리멘트를 사용하는 경우
         <!ELEMENT  exam  ANY>

 (3) xml에서 DTD의 연결
     <!DOCTYPE  (rootElement 이름)  PUBLIC/SYSTEM  "DTD문서의 경로">

     * public : 공인된 DTD문서
     * system : 공인되지 않은 문서

  (4) 속성(attribute) 선언 문법
      <!ATTLIST  (엘리멘트명)  (속성명)  (속성유형)  (디폴트선언)>

     1)  디폴트선언
       * #IMPLIED : 엘리멘트 작성시 속성을 생략할 수 있다.
       * #REQUIRED : 엘리멘트 작성시 반드시 기술한다.
       * #FIXED : 속성값으로 지정한 문자만 사용한다.
       * "디폴트값" : 속성이 생략되었을 경우, 디폴트값을 기본값으로 사용한다.

     2) 속성 유형
       a. CDATA : 문자열 데이터
       b. ENUMERATION : 나열한 값 중 하나를 사용한다...
       c. ID : 유일한 데이터를 사용한다. (primary key)
       d. IDREF : ID가 가지고 있는 값만 사용한다.
       e. NMTOKEN : 공백이 없는 문자열

       f. NOTATION : DTD에 선언된 NOTATION명만 사용한다.
       g. ENTITY : DTD에 선언된 ENTITY명만 사용한다.

5. Entity
  (1) 정의
      --> Xml 문서의 물리적 저장 단위

  (2) 분류
    1) 물리적 저장단위
       * 내부 : DTD 내에 Entity 존재
       * 외부 : 파일 형태로 존재할 경우

    2) 사용하는 곳
       * 일반 : xml에서 사용
       * 파라미터 : DTD에서 사용

    3) 해석 여부
       * 파스드 : 해석 가능
       * 언파스드 : 해석 불가능

  (3) 내부 일반 파스드 Entity
     선언 : <!ENITYT  (엔티티명)  "(대치할 문자열)">
     사용 : &(엔티티명);

  (4) 외부 일반 파스드 Entity
     선언 : <!ENTITY  (엔티티명)  SYSTEM  "(파일의 경로)">
     사용 : &(엔티티명);

  (5) 내부 파라미터 Entity
     선언 : <!ENTITY  %  (엔티티명)  "(대치할 문자열)">
     사용 : %(엔티티명);

  (6) 외부 파라미터 Entity
     선언 : <!ENTITY  %  (엔티티명)  SYSTEM  "(파일의 경로)">
     사용 : %(엔티티명);

6. 컨디셔널 섹션
  --> 조건을 확인해서 관련된 구조를 적용하는 방법
  
  <![INCLUDE[
        적용할 DTD 내용
  ]]>
  <![IGNORE[
        적용하지 않을 DTD 내용
   ]]>
