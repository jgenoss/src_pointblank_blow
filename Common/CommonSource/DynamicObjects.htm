2. Object Physix State

    m_PhsixState

    1) I3OBJ_PHYSIX_STATE_DYNAMIC

        물리가 적용되는 오브젝트

        CS에서만 계산하며 CN은 Position, Rotation 값을 패킷으로 받는다.

    2) I3OBJ_PHYSIX_STATE_KINEMATIC

        Animation을 하는 오브젝트, 게임에서 관리한다.

    3) I3OBJ_PHYSIX_STATE_STATIC

        월드에 고정된 오브젝트

    4) I3OBJ_PHYSIX_STATE_NONE

        아무런 기능을 하지 않는다.

 

3.  Object Element State

    m_State ;

3. Object Element Style

    m_Style ;

    1) I3OBJ_ELEMENT_STYLE_NORMAL

        일반적인 형태를 갖는다.

    2) I3OBJ_ELEMENT_STYLE_BREAK

        HP를 가지며 각 HP에 따른 단계별 외형을 가질 수 있다.

         - Step : 여러 스텝에 따른 HP량과 외형(SceneObj)을 세팅. (최종 단계량을 결정할 필요 있음.)

         - Removable : HP가 0이면 사라지는지에 대한 세팅

        * Variable *

        m_CurHP;    현재 HP

        m_pCurObj; 현재 오브젝트 외형

        m_BreakStepList ; 각 스텝에 대한 정보 리스트

        I3OBJ_ELEMENTINFO_BREAKSTEP

        {

            m_HP ; Hitpoint

            m_pNextObj ; HP 이하의 값이 되면 설정된 오브젝트로 교체

        }

 

    3) I3OBJ_ELEMENT_STYLE_TRIGGER 

     -> Level Primitve와 비슷한 기능을 갖지만 Dynamic or Kinematic 이 될 수 있는 오브젝트이다.

        발동에 의한 상태변화를 갖는 오브젝트( 발동 타입, 발생 이벤트를 제어해야된다..)

        - Active Type : Hit, On/Off(키입력에 따른 이벤트가 될것), Pickup, Break ..etc

        - Event Type : Break, Change SceneObj, Attach, SendMessage,

        * Variable *

        m_ActiveType ; Trigger 발동 타입

            enum I3TRIGGER_ACTIVE_TYPE {

                I3TRIGGER_ACTIVE_TYPE_NONE,

                I3TRIGGER_ACTIVE_TYPE_ON,                           // 다른 이벤트에 따라 bool값으로 On되는 경우

                I3TRIGGER_ACTIVE_TYPE_COLLISION,               //  Collision에 의한 경우

                I3TRIGGER_ACTIVE_TYPE_BREAK,                     // 자신이 파괴된 경우

            }

        m_EventType ; Trigger action 타입

            enum I3TRIGGER_ACTION_TYPE {

                I3TRIGGER_ACTION_TYPE_NONE,

                I3TRIGGER_ACTION_TYPE_SWAP,                      // 자신의 외형을 바꾼다

                I3TRIGGER_ACTION_TYPE_MESSAGE,

                I3TRIGGER_ACTION_TYPE_HIDE,

            }

 

    4) I3OBJ_ELEMENT_STYLE_PATH     ->엘리베이터, 크레인 등

        Path를 따라 가는 Object. DynamicState는 Kinematic만 해당된다.

        - Pickup path : 하나의 패스를 지정해야된다.

        - Run path : Auto, On, Off    ( 패스를 따를지에 대한 끄고, 키는 플래그 필요. )

        m_pPath ;
