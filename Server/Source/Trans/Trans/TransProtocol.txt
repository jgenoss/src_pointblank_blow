Auth Protocol Draft
2007.05.09 / 류성훈 praptor@zepetto.com


Overview

기본적으로 AuthServer에서 GameServer로 보내는 요청이다.
AuthServer는 GSM, Web으로 부터 요청을 받는다.
AuthServer와 DB만으로 해결 가능한 요청에 대해선 protocol이 필요없다.
GSM의 요청엔 처리 후 결과를 알려 줄 수 없다.
Web의 요청중 Get 계열은 보내줄 팻킷량 파악이 필요하다. (매우 클듯)


Protocol

PROTOCOL_ACCOUNT_KICK_REQ
PROTOCOL_ACCOUNT_KICK_ACK
	User를 kick
	Req. GSM, Web
	Not Ack.
	> UID				4	bytes
	> GUSID				16	bytes
	> Reason			1	byte
	< none
	
PROTOCOL_PUSH_ITEM_REQ
PROTOCOL_PUSH_ITEM_ACK
	Item 구입
	Req. GSM
	Not Ack.
	> UID				4	bytes
	> Account			100	bytes
	> ItemID			4	bytes
	> ItemName			100 bytes
	> ItemCount			1	byte
	< none
	Client 통보 여부와 범위에 따라 보내는 패킷을 조정해야한다

PROTOCOL_JOIN_GAME_REQ
PROTOCOL_JOIN_GAME_ACK
	Join Game
	Req. GSM
	Not Ack.
	> UID				4	bytes
	> Account			100	bytes
	< none
	GSM에서 보내주는 내용으로 용도 불명

PROTOCOL_PUSH_PRESENT_ITEM_REQ
PROTOCOL_PUSH_PRESENT_ITEM_ACK
	선물 받기
	Req. GSM
	Not Ack.
	> UID				4	bytes
	> Account			100	bytes
	> GuessWho			100	bytes	// 선물 준 사람
	> ItemID			4	bytes
	> ItemName			100	bytes
	> ItemCount			1	bytes
	Client 통보 여부와 범위에 따라 보내는 패킷을 조정해야한다

PROTOCOL_ALARM_COIN_REQ
PROTOCOL_ALARM_COIN_ACK
	Alram Coin
	Req. GSM
	Not Ack.
	> GUSID				16	bytes
	> PaidCoin			4	bytes	// 캐쉬머니?
	> FreeCoin			4	bytes	// 게임머니?
	UBS에서 GSM으로 넘어오며 추가된 내용으로 용도 불명

PROTOCOL_BREAK_ROOM_REQ
PROTOCOL_BREAK_ROOM_ACK
	방 깨기
	Req. Web
	> 아이디
	> 방번호
	< 성공여부
	Web에서 User 중심으로 방 번호를 찾는다
	방 번호만을 알려준다면 서버(+채널) 위치도 알려줘야한다
